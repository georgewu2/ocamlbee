The Rules

Rules of the Bee

  • Players will first hear the problem statement. For example: 
    Write a function that takes two
integers and returns their sum.
  • Players will alternate writing the code, each specifying one character at a time.
  • Players are not allowed to talk to one another after the problem statement is given. Communication can only be through code written.
  • Writing comments in code is allowed. Your code will only be tested for functionality in the end.
  • Players will have 4 minutes per problem to write a complete function, starting from when the problem is completely read.
  • Players can ask for problem clarifications ONLY at any time. (No communication about algorithms or library functions.)
  • Players can use the standard keyboard and the “Clear line” option which will delete the entire line that the two teams are working on.
  • For these specific keywords, use the following phrases:
    • ‘\t’ : “TAB”
    • ‘\n’ : “NEWLINE”
    • ‘clear’ : ‘CLEAR LINE’

Functions

  • Function must be named "camel". Example solution to the problem above: 
    let camel x y = x + y
    • In other words... "l, e, t, space, c, a, m, e, l, space, equals, ..."
  • Watch out for runtime errors (division by zero, interpretation error, etc.)
  • Your code MUST compile using OCaml 4.0.
  • You may use ANY module in the OCaml Standard Library. This includes List, Set, Map, String, etc.
  • Your code will be tested on a small set of test cases for verifcation. However all decisions by the judges will be final.

Teams

  • Each team must have 2 members.
  • If you don't have a partner, sign up anyway and we will pair you up!

Scoring

  • Teams who spell out a correct solution in a round will move to the next round.
  • If no teams correctly spell out a solution in a round, all teams from that round will move on.
  • When there are no more than 3 teams left and only one team correctly spells a solution, that team will be the Champion.
  • If there are more than 3 teams left and only one team moves to the next round, that team must solve an additional problem to be crowned Champion. If they are unsuccessful, all teams from the previous round will remain.

Tips

  • Use comments to communicate.
  • Make use of clear lines!
  • Bring an identical twin.

Misc. / Questions

  • These rules are meant as general guidelines. Finding technicalities is not in good nature of the Bee.
